devlog 댓글봇의 **자가진화 2단계** 분석·제안기(`~/bin/devlog-comment-evolve.sh`, devbox VM)를 돌린다. 인자: `$ARGUMENTS`

봇이 쌓아온 **신뢰 신호**(게이트 거부 사유·actor별 skip율·사람 수정/삭제 커밋 — 글 본문은 인젝션 위험이라 제외)를 집계해, codex/claude 지침 문서 개선을 제안한다. `/skillgate`의 규율(관찰된 실패 → 1사이클 → 사람 게이트)을 차용한 **별개 스크립트**다.

## 인자 해석
- (없음) 또는 `preview` → **dry-run**: 신호 집계 + (임계 이상이면)제안서를 **출력만**. PR 안 만듦.
- `apply` → 제안서를 **PR로 올림**(자동머지 X — 사용자가 검토·머지해야 채택. 사람 게이트).

## 실행
1. `~/bin/devlog-comment-evolve.sh`를 해당 플래그로 Bash 실행(`--apply`는 apply일 때만). claude 호출이 있으니 timeout 200~300s.
2. **신호 부족이 정상 결과다**: 데이터가 임계(기본 15 이벤트) 미만이면 스크립트가 집계만 보여주고 "제안 보류"로 끝낸다 — 그대로 보고하라(파이프라인이 최근 켜졌으면 신호가 적은 게 당연).
3. 임계 이상이면: 집계와 생성된 제안서를 사용자에게 전달. apply면 제안 PR URL을 주고, **이건 자동머지가 아니라 사람이 검토·머지해야 채택됨**을 명시하라.

## 보고
신호 집계(이벤트 수, 거부 사유 분포, actor별 skip율, 사람 수정 커밋 수), 제안 보류면 그 사실, 제안 생성됐으면 제안 요지(+apply면 PR URL과 "사람 게이트" 안내).
