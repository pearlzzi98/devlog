## 제안
## devlog 댓글봇 지침 개선 제안서

**결론: 제안 없음 — 신호가 특정 패턴을 가리키지 않음.**

근거를 아래에 정리한다. 규칙상(신호 없는 추측 변경 금지) 어떤 문구도 바꾸지 않는다.

### 왜 제안할 게 없나 (신호 판독)

| 신호 | 값 | 무엇을 가리키나 |
|---|---|---|
| 사람 수정/삭제 커밋 | **0 / 114** | 게시된 댓글에 대한 **부정 신호가 0**. 내용 정책(§4/§3)을 손댈 근거 없음 — 고칠 결함이 관측되지 않음. |
| skip율 (claude) | 18% (10/55) | actor 간 차이 거의 없음 → |
| skip율 (codex) | 16% (10/49) | **actor별 분기 없음** → 한쪽 지침만 조이거나 푸는 변경 불가. |
| skip_model 총량 | 17.5% (20/114) | 목표치·skip **사유 분포**가 없어 "많다/적다" 판정 불가. skip이 맞았는지 틀렸는지 데이터 없음. |

핵심: 변경을 정당화하려면 둘 중 하나가 필요한데 **둘 다 신호에 없다.**
- *조이는 방향* → "wrote 중 품질 나쁜 게 있었다"는 증거 필요 → 사람 수정/삭제=0이 정면으로 반박.
- *푸는 방향* → "skip이 과했다(놓친 글)"는 증거 필요 → skip 사유·오탐 데이터 없음.

balanced skip + zero human correction 조합은 "현 지침이 깨졌다"가 아니라 "관측 가능한 결함이 없다"에 해당한다. 여기서 §4·§5(내용/스킵 기준)를 건드리는 건 신호 없는 추측이다.

### 채택과 무관한 메모 (지침 변경 아님 — 다음 사이클을 위한 계측 제안)

지금 집계로는 어떤 지침 줄도 결함과 연결할 수 없다. 다음 라운드에 *실제* 제안을 내려면 아래 신호가 필요하다(코드/수집 쪽, 지침 문서와 무관):

1. **skip_model의 사유 분포** — `COMMENT_SKIPPED:`/`REPLY_SKIPPED:` 뒤 사유를 버킷팅(본문 짧음 / 회고 없음 / 민감정보 / 가치 없음). 어느 §5/§4 줄이 작동/오작동하는지 그제서야 보인다.
2. **skip 오탐 라벨** — skip된 글 표본에 "사실은 썼어야 함" 사람 라벨 소수. 없으면 skip율은 해석 불가 숫자.
3. **wrote 품질 약신호** — 사람 수정/삭제가 0이라 신호가 죽어 있음. 가벼운 👍/👎나 길이·반복 자동지표라도 있어야 내용 정책을 데이터로 움직일 수 있다.

이 셋 중 하나라도 들어오면 구체적 문구/few-shot 제안으로 전환 가능하다. 그 전까지는 **현 지침 유지가 신호에 부합하는 선택**이다.
