## 제안
## 개선 제안서 (devlog 댓글봇 지침)

**제안 없음: 신호가 특정 패턴을 가리키지 않음.**

근거 — 규칙상 "신호에 없는 추측 변경 금지"라서, 이번 집계가 어떤 지침 문구와 연결되는지 점검한 결과:

- **집계가 수치뿐이고 내용 신호가 없음.** outcome은 `wrote/skip_model` 카운트만 있고, 어떤 댓글이 왜 부적절했는지(과장·인젝션 추종·민감값 노출·동의만 등) 패턴 분해가 없다. 특정 절(§4 내용정책, §5 skip 기준 등)을 가리키는 근거가 없다.
- **사람 수정/삭제 = 0** → 발행물에 대한 **부정 품질 신호가 0건**. 고쳐야 할 출력 패턴이 관측되지 않았다는 뜻이라, 어느 지침을 조일/풀 방향도 신호가 주지 않는다.
- **skip율 차이가 무의미**(codex 14% vs claude 17%, n=11 vs 13). 3pp 차이·소표본이라 두 지침 문서의 **특정 차이로 귀속할 수 없다**. 게다가 두 역할(최초 댓글/답글)은 skip 기준이 원래 다르게 설계돼 있어, 이 정도 차이는 설명 불필요.
- skip율 16%(24/151) 자체도 **목표·기준선이 없어** 높다/낮다 판단 불가 → 임계 조정 제안의 근거가 없다.

→ 바꿀 문구·few-shot를 신호에 묶을 수 없으므로 변경을 제안하지 않는다.

**다음 사이클에 더 나은 제안을 내려면 필요한 신호**(참고, 이번 출력의 변경 제안 아님):
- skip 사유 분포(`COMMENT_SKIPPED:` / `REPLY_SKIPPED:` 사유별 카운트) — 짧음/무내용 vs 민감정보 vs 가치없음 구분
- 코드 검증 단계의 **리젝트 사유별 카운트**(길이·민감값·AI자기언급·과장) — 어느 §4 금지항목이 실제로 걸리는지
- 사람 수정이 0이라도, 사람이 **남긴 반응/추천 여부** 같은 약한 품질 프록시

이 중 하나라도 패턴이 잡히면 그때 해당 절에 최소 변경을 제안할 수 있다.
